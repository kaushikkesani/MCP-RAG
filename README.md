# RAG MCP 101

> Exposing a local RAG pipeline as an MCP tool — built with Python, sentence-transformers, ChromaDB, Ollama and MCP.

---

## What this project does

RAG MCP 101 wraps a fully local RAG (Retrieval-Augmented Generation) system behind an MCP (Model Context Protocol) server. Any MCP-compatible AI agent can call the `query_documents` tool to ask questions about an ingested PDF — the server handles embedding, vector search, and LLM generation entirely on your machine. No API keys. No cloud. No data leaving your laptop.

---

## Project structure

```
rag_mcp_101/
├── rag_core/
│   ├── __init__.py
│   ├── chunker.py        # splits text into overlapping chunks
│   ├── embedder.py       # converts text to vectors (all-MiniLM-L6-v2)
│   ├── pdf_reader.py     # extracts text from PDF using pypdf
│   └── retriever.py      # ChromaDB storage, retrieval, Ollama call
├── chromadb_store/       # ← generated by ingest.py (do not edit)
├── ingest.py             # one-time ingestion pipeline
├── mcp_server.py         # MCP server — exposes query_documents tool
├── your_document.pdf     # ← swap in your own PDF
└── venv/
```

## Setup

**1 — Clone or copy the project and create a virtual environment**
```bash
cd rag_mcp_101
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux
```

**2 — Install dependencies**
```bash
pip install sentence-transformers chromadb pypdf ollama mcp fastmcp
```

**3 — Pull the Llama 3 model (~4GB, downloads once)**
```bash
ollama pull llama3
```

## Running the system

### Step 1 — Ingest your PDF (run once)

Drop your PDF into the project folder, update `PDF_PATH` at the top of `ingest.py`, then run:

```bash
python ingest.py
```

Expected output:
```
[1/3] Reading PDF: your_document.pdf
      Done — XXXXX words extracted
[2/3] Chunking text (size=200, overlap=50)
      Done — XXXX chunks created
[3/3] Embedding and storing in ChromaDB at './chromadb_store'
      Stored XXXX chunks to ./chromadb_store
Done. ChromaDB is saved to disk and ready for mcp_server.py
```

> Re-run `ingest.py` only if your source document changes. The `chromadb_store/` folder persists between server restarts — no re-embedding needed.

---

### Step 2 — Start the MCP server

```bash
python mcp_server.py
```

Expected output:
```
Starting RAG MCP 101 server on http://127.0.0.1:8000
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
```

Leave this terminal running.

---

### Step 3 — Open MCP Inspector (in a second terminal)

```bash
npx @modelcontextprotocol/inspector
```

When the browser opens:
- **Transport Type** → `SSE`
- **URL** → `http://127.0.0.1:8000/sse`
- Click **Connect**

You should see a green **Connected** dot and the `query_documents` tool listed.

---

### Step 4 — Ask a question

Click `query_documents`, type your question, click **Run Tool**. The full pipeline runs — embedding → ChromaDB retrieval → Ollama generation — and the answer appears under **Tool Result: Success**.

---

## How it works

```
INGESTION (once)
PDF → pdf_reader.py → chunker.py → embedder.py → ChromaDB (disk)

QUERY (every question)
question → mcp_server.py → embedder.py → ChromaDB search → retriever.py → Ollama → answer
```

The MCP server exposes one tool:

| Tool | Input | Description |
|------|-------|-------------|
| `query_documents` | `question: str` | Searches the knowledge base and returns a grounded answer |

## Configuration

All tuneable settings are at the top of each file:

| File | Setting | Default | Description |
|------|---------|---------|-------------|
| `ingest.py` | `PDF_PATH` | `"your_document.pdf"` | Path to your source PDF |
| `ingest.py` | `CHUNK_SIZE` | `200` | Words per chunk |
| `ingest.py` | `OVERLAP` | `50` | Overlapping words between chunks |
| `ingest.py` | `PERSIST_PATH` | `"./chromadb_store"` | Where ChromaDB saves to disk |
| `mcp_server.py` | `PERSIST_PATH` | `"./chromadb_store"` | Must match ingest.py |
| `mcp_server.py` | `port` | `8000` | HTTP port for SSE transport |
| `rag_core/retriever.py` | `n_results` | `3` | Number of chunks retrieved per query |

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `ModuleNotFoundError` | Make sure venv is active: `venv\Scripts\activate` |
| ChromaDB appends on re-run | Delete `chromadb_store/` folder then re-run `ingest.py` |
| MCP Inspector times out | Use SSE transport, not STDIO — see Step 3 above |
| Ollama not responding | Run `ollama ps` to check if it's active; start with `ollama serve` |
| `embeddings.position_ids UNEXPECTED` warning | Harmless — safe to ignore |
| `HF_TOKEN unauthenticated` warning | Harmless — model is public, no token needed |
